# Cut2Kit Bootstrap Handoff

## Status

The first Cut2Kit vertical slice is mostly implemented.

The repo now has:

- shared Cut2Kit schemas/contracts in `packages/contracts/src/cut2kit.ts`
- shared prompt/summary helpers in `packages/shared/src/cut2kit.ts`
- deterministic server-side project scan, settings validation, manifest derivation, and placeholder NC generation in `apps/server/src/cut2kit/`
- WebSocket RPC wiring for `cut2kit.inspectProject` and `cut2kit.generateOutputs`
- a project workspace route at `apps/web/src/routes/_chat.project.$projectId.tsx`
- a central project view in `apps/web/src/components/Cut2KitProjectView.tsx`
- a sidebar project explorer in `apps/web/src/components/sidebar/Cut2KitProjectExplorer.tsx`
- sample project fixtures under `examples/`
- bootstrap reconnaissance notes in `docs/cut2kit-bootstrap-notes.md`
- top-level run docs in `README.md`

## What Was Implemented

### Domain/contracts

- Added explicit schema/types for:
  - `Cut2KitProject`
  - `ProjectFileRecord`
  - `Cut2KitSettings`
  - `FramingRuleSet`
  - `PanelManifest`
  - `NestManifest`
  - `QueueManifest`
  - `NCJobRecord`
- Added RPC/native API contracts for:
  - `cut2kit.inspectProject`
  - `cut2kit.generateOutputs`

### Server

- Added deterministic Cut2Kit service layer in `apps/server/src/cut2kit/Layers/Cut2KitProjects.ts`
- Current behavior:
  - recursively scans project folders
  - detects `cut2kit.settings.json`
  - classifies DXFs/settings/manifests/NC/reference files
  - validates settings JSON via schema
  - derives source docs, panel candidates, panel/nest/queue manifests
  - writes deterministic placeholder outputs under `output/manifests` and `output/nc`
  - blocks generation if validation errors exist
- Registered service in:
  - `apps/server/src/server.ts`
  - `apps/server/src/ws.ts`

### Web

- Added project query helper in `apps/web/src/lib/cut2kitReactQuery.ts`
- Added project workspace route and navigation wiring
- Changed the project-open flow in the sidebar so a newly added project lands in a project workspace view instead of forcing the user into a thread immediately
- Added Cut to Kit Agent preparation flow:
  - builds a project-context prompt
  - opens/reuses a draft Codex thread
  - sets `runtimeMode: "approval-required"`
  - relies on existing approval/diff flow for any edits

### Fixtures/docs

- Added:
  - `examples/cut2kit.settings.example.json`
  - `examples/prefab-demo-project/cut2kit.settings.json`
  - placeholder DXFs under `examples/prefab-demo-project/`
- Wrote:
  - `docs/cut2kit-bootstrap-notes.md`
  - updated `README.md`
